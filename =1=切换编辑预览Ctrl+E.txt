### 第一步：下载与安装 Obsidian

1. **访问官网**：打开浏览器，输入 `https://obsidian.md`。
2. **下载 Windows 版**：首页会自动识别你的系统，点击 **Download for Windows**（64 位安装包 `.exe`）。
3. **安装**：
   - 双击运行安装程序。
   - **建议勾选** “Create a desktop shortcut”（创建桌面快捷方式）。
   - 安装路径可以保持默认 `C:\Users\你的用户名\AppData\Local\Obsidian`，也可以自行修改，**不影响**你的知识库数据位置（数据是独立存放的）。

---

### 第二步：创建你的第一个知识库（Vault）

安装完成后打开 Obsidian，会弹出欢迎界面：

1. 点击 **Create new vault**（创建新库）。
2. **Vault name** 填写：`MyKnowledgeBase`（这个名字仅作为显示标识）。
3. **Location** 点击 **Browse**，浏览并选择路径：`Y:\i\aa\MyKnowledgeBase`。
4. 点击 **Create**。

此时 Obsidian 会打开一个空白的库，界面如下：

- 左侧：文件列表（目前为空）。
- 中间：笔记编辑区（显示帮助文档，可以关掉）。
- 右侧：默认无面板，后续可开启大纲/反向链接。

### 第六步：接入 Git 版本控制（关键一步）

**前置条件**：你的电脑已安装 Git（`git --version` 检查）。

1. **初始化仓库**：
   - 打开命令行（Win + R → `cmd`），进入库路径：
     ```cmd
     Y:
     cd Y:\i\aa\MyKnowledgeBase
     git init
     ```
   - 创建 `.gitignore` 文件（用记事本创建，内容如下）：
     ```gitignore
     # Obsidian 工作区文件
     .obsidian/workspace.json
     .obsidian/workspace-mobile.json
     .trash/
     ```
   - 执行首次提交：
     ```cmd
     git add .
     git commit -m "init: Obsidian 知识库初始化"
     ```

2. **配置 Obsidian Git 插件**：
   - 在 Obsidian 设置 → **Obsidian Git** 插件中：
     - **Vault backup interval (minutes)**：设置为 `30`（每30分钟自动提交一次）。
     - **Auto pull on startup**：开启（打开 Obsidian 时自动拉取最新）。
     - **Commit message**：可留空，或填 `vault backup: {{date}}`。
     - **Pull changes before push**：开启（防止冲突）。
   - 点击左侧边栏出现的 **Open Git Source Control** 按钮，点击 **Initialize**（如果提示未初始化）。
   - 之后你可以手动点击 **Commit** 并 **Push**，也可以让它后台自动运行。

### 第七步：部署 Python 辅助脚本（为 AI 自动化铺路）

在 `90_System` 文件夹下新建 `md_helper.py`，内容如下（增强版，支持统计字数、合并输出）：

```python
# md_helper.py - Markdown 知识库辅助脚本
# 用法示例: 
# python md_helper.py count                # 统计字数
# python md_helper.py merge 20_Articles    # 合并某个目录下所有md文件

import os
import sys
from pathlib import Path

VAULT_ROOT = Path(__file__).parent.parent  # 自动定位到 MyKnowledgeBase 根目录

def count_all_words():
    total_chars = 0
    for md_file in VAULT_ROOT.rglob("*.md"):
        try:
            with open(md_file, 'r', encoding='utf-8') as f:
                total_chars += len(f.read())
        except:
            pass
    print(f"知识库总字符数: {total_chars:,}")

def merge_folder(folder_name):
    target_dir = VAULT_ROOT / folder_name
    if not target_dir.exists():
        print(f"目录不存在: {target_dir}")
        return
    
    output_file = VAULT_ROOT / f"合并_{folder_name}.md"
    with open(output_file, 'w', encoding='utf-8') as out:
        for md_file in sorted(target_dir.glob("*.md")):
            out.write(f"\n\n---\n## 来源: {md_file.name}\n---\n\n")
            with open(md_file, 'r', encoding='utf-8') as f:
                out.write(f.read())
    print(f"合并完成 -> {output_file}")

if __name__ == "__main__":
    if len(sys.argv) < 2:
        print("请提供命令: count 或 merge 文件夹名")
    elif sys.argv[1] == "count":
        count_all_words()
    elif sys.argv[1] == "merge" and len(sys.argv) > 2:
        merge_folder(sys.argv[2])
    else:
        print("未知命令")
```

**如何测试运行：**

1. 打开终端，进入 `Y:\i\aa\MyKnowledgeBase\90_System`。
2. 执行 `python md_helper.py count`。
3. 你会看到类似输出：`知识库总字符数: 1,234`。

这个脚本未来可以扩展：比如接入 OpenAI API 自动摘要、根据 `tags` 分类生成索引页、检测断链等等，都是你实践“工具自动化”的绝佳练手项目。

### 最后提醒（关于数据安全）

`Y:\i\aa\MyKnowledgeBase` 这个路径如果在本地硬盘或可靠 NAS 上，配合 Git 基本万无一失。如果 `Y:` 是移动硬盘/U盘，**务必注意**：Obsidian 缓存文件 `.obsidian` 较大，频繁读写可能影响寿命，但影响甚微；更重要的是 Git 仓库的完整性，建议至少每周 `git push` 到远程仓库（GitHub/Gitee 私有库）一次，实现 **3-2-1 备份**。
